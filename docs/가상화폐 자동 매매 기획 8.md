# 가상화폐 자동 매매 기획 v8.0 — Hybrid LLM Trading

## 1. 개요

v8.0은 기계적 기술 분석과 LLM(Large Language Model)의 인간적 판단을 결합한 
하이브리드 트레이딩 시스템입니다.

### 1.1 v8.0 목표

1. **매매 빈도 증가**: v7.0 대비 진입 조건 완화로 적절한 매매 기회 확보
2. **승률 향상**: LLM 검증을 통한 고품질 신호 필터링
3. **손실 복구**: 30만원 → 2만원 손실을 회복하기 위한 공격적이되 계산된 접근
4. **자동화 극대화**: 인간 개입 없이 24/7 자율 운영

### 1.2 핵심 철학

```
기계적 신호 생성 → LLM 검증 → Confluence Score 계산 → 동의 시에만 진입
```

## 3. LLM Advisor 상세

### 3.1 시스템 프롬프트 요약

LLM은 15년 경험의 암호화폐 트레이더 역할을 수행:

- **자본 보존 최우선**: 거래당 리스크 1-2%
- **컨플루언스 요구**: 3개 이상 지표 동의 필요
- **보수적 기본값**: 불확실 시 HOLD
- **리스크/리워드**: 최소 1:2 유지

### 3.2 LLM 입력 데이터

```python
{
    "market": "KRW-BTC",
    "current_price": 140000000,
    "price_change_1h": -0.5%,
```

## 4. Confluence Score 계산

### 4.1 구성 요소

| 요소 | 가중치 | 범위 | 설명 |
|------|--------|------|------|
| Trend Score | 35% | -1 ~ +1 | EMA 정배열, 가격 위치, ADX 강도 |
| Momentum Score | 30% | -1 ~ +1 | RSI, MACD, BB 위치 |
| Volume Score | 20% | -1 ~ +1 | 거래량 강도, 추세 확인 |
| Pattern Score | 15% | -1 ~ +1 | 캔들 패턴, Higher Lows |
| LLM Modifier | ±15% | -0.15 ~ +0.15 | LLM 동의/반대 보너스 |

### 4.2 진입 조건

```python
# 기본 임계값
MIN_CONFLUENCE_SCORE = 0.55

# 진입 조건
if confluence_score >= MIN_CONFLUENCE_SCORE:
    if llm_signal.action == "BUY":
        # LLM 동의 → 신뢰도 +15%
        execute_buy()
    elif llm_signal.action == "HOLD":
        # LLM 중립 → 기계적 신호만으로 진입
        execute_buy_with_reduced_size()
    else:
        # LLM 반대 → 진입 보류
        hold()
```

## 5. 시장 센티먼트 분석

### 5.1 김치 프리미엄

- **계산**: (업비트 가격 - 바이낸스 가격×환율) / 바이낸스 가격
- **해석**:
  - > +5%: 과열 경고 → 신규 매수 억제
  - +1% ~ +5%: 정상 범위
  - < -1%: 할인 구간 → 매수 기회

### 5.2 Fear & Greed Index

- **출처**: alternative.me API
- **해석**:
  - 0-25: Extreme Fear → 매수 기회 (+15% 신뢰도)
  - 25-45: Fear
  - 45-55: Neutral
  - 55-75: Greed
  - 75-100: Extreme Greed → 매수 억제 (-15% 신뢰도)

## 6. 리스크 관리

### 6.1 포지션 사이징

```python
# ATR 기반 1% 리스크 사이징
risk_amount = equity × 0.01  # 자본의 1%
stop_distance = atr × stop_multiplier
position_size = risk_amount / stop_distance
```
