# v6.0 보수적 자본 보존 전략 전면 개정

> **발단:** 30만원 → ~4만원 (87% 손실)
> **개정일:** 2026-02-26

## 손실 원인 분석 (9가지)

### 1. 4개 독립 매매 루프 동시 매수
- `run_cycle` (3레이어 엔진), `run_tick_cycle` (ML 단독), `run_pump_detection_loop` (웹소켓), `run_emergency_check` (긴급)
- 각 루프가 독립적으로 BUY 신호를 생성하여 과매매 발생

### 2. 공격적 매매 모드 활성화
- `aggressive_trading_mode=True` → ML 신호만으로 50% 포지션 진입
- LLM 검증 없이 거래하여 거짓 신호에 매수

### 3. 투자 비율 과다
- 신뢰도 90%에서 자산의 40% 투입
- 4개 루프가 동시에 매수하면 자산 100% 이상 노출

### 4. 역방향 손익비
- 손절 -3% vs 익절 +2% → 승률 60% 이상이어야 본전
- ATR 기반 동적 SL/TP가 -4%까지 허용 → 리스크 확대

### 5. 일일 손실 한도 부재
- 연속 손실에도 계속 거래 → 하루에 전체 자산의 20%+ 손실 가능

### 6. 긴급 매수 = FOMO 매수
- 급등 감지 시 자동 매수 → 고점에서 물림

### 7. 페르소나 BUY 오버라이드
- 래리 윌리엄스 페르소나가 HOLD → BUY로 변환
- 실제로는 약한 신호를 매수로 전환하는 효과

### 8. 손절 후 쿨다운 없음
- 손절 직후 다음 사이클에서 즉시 재진입 → 추가 손실

### 9. 돌파 전략 과민 반응
- RSI 50-80 (너무 넓음), 거래량 2.5배 (낮은 임계값)
- 과다한 BUY 신호 생성

### trading.py 변경
1. **일일 손익 추적 시스템**: `_daily_pnl_krw`, `_daily_trade_count`, `_last_loss_time` 전역 변수
2. **`_is_trading_allowed()` 안전장치**: 일일 손실 한도, 쿨다운, 거래 횟수 체크
3. **포지션 관리 강화**: -1.5% 즉시 손절, -2.5% Hard Stop, 3단계 트레일링 스탑
4. **페르소나 BUY 오버라이드 제거**: SELL 거부권만 유지
5. **투자 비율 축소**: 최고 25% (기존 40%)
6. **SL/TP 개선**: 1.5% SL / 3% TP 고정, ATR 보정은 좁히기만 가능
7. **긴급 매수 차단**: emergency_check에서 BUY 실행 안 함
8. **tick_cycle/pump_detection**: 설정으로 비활성화
